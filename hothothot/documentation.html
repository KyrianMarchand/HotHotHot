Les différents outils utilisés

Pour la programmation, nous avons utilisé le logiciel Visual Studio Code.
Pour le partage des fichiers, nous avons utilisé GitHub ainsi que Google Drive.
Pour l'hébergement du site, nous avons utilisé un hébergeur gratuit, Alwaysdata.
Pour la réalisation du diagramme de Gantt, nous avons utilisé le site web Lucidchart.



L'organisation

Gestion du projet

Pour commencer, nous avons mis en place un Repository Github pour se partager le code plus facilement entre nous. Nous avons également mis en place un Google Document pour pouvoir rédiger à plusieurs la documentation au brouillon avant de la mettre en place sur le site.



Organisation dans le temps

Nous avons d'abord appris a utiliser les capteurs a partir du websocket et a les afficher. Nous avon ensuite créer un tableau d'historique que nous avons transformer en graphique avec l'aide de chartjs dans lequel nous affichons la valeur des capteurs qui avance avec le temps. En paralèlle nous avons mis en place le certificat SSL ainsi que le service worker et la PWA. Après cela nous avons mis en place les alerte et notifications.



Principales Difficultés, Solutions apportées

La difficulté la plus importante a été de mettre le graphe sur une page différente de celle des capteurs. Cependant, mais si nous avions fait cela les données ce serait réinitialisées à chaque changement de page.
Pour résoudre ce problème, nous avons donc choisi de pouvoir faire "disparaître" et "réapparaître" les données.

La deuxième difficulté a été le Web Socket qui marchait une fois sur trois, ce qui nous oblige à actualiser plusieurs fois la page avant qu'elle ne marche.
Nous n'avons malheureusement pas pu résoudre ce problème qui ne dépend pas de nous.



Si vous deviez recommencer ce projet, que changeriez vous à vos pratiques ?

Si nous devions recommencer le projet, nous pensons que l'on devrait plus s'appuyer sur les outils dont on dispose, comme par exemple GitHub que nous n'avons pas assez utilisé. Il faudrait également travailler de façon plus régulière afin de ne pas faire beaucoup de code d'un coup, et plus rien du tout pendant certaines périodes.
